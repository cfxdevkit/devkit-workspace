# Brand System

## Brand Position

Conflux DevKit is infrastructure for builders. The identity communicates:
- speed to first transaction
- confidence and control
- real local chain execution
- AI-assisted developer workflows
- a modern editor-first experience

## Brand Statement

Conflux DevKit turns local Conflux development into a repeatable, resumable workflow.
Instead of hours assembling a node, wallet, contracts, DEX, and AI tool chain, developers open one workspace and start building.

## Taglines

Primary: **One command. Full Conflux.**

Alternatives:
- Real Conflux. Local control.
- Build on Conflux without setup drag.
- Your Conflux dev stack, already running.

## Visual Direction

The system should look like an intentional developer tool:
- dark graphite foundation with subtle blue undertones
- electric cyan as the primary action color
- bright Conflux-adjacent orange as the accent and energy color
- subtle dot grids and ambient glow zones instead of heavy line grids
- monospace headlines for tool credibility and sans-serif body text for readability

## Type Guidance

- **Headlines**: JetBrains Mono, IBM Plex Mono, ui-monospace, monospace
- **Body**: Inter, Space Grotesk, Segoe UI, sans-serif
- Use uppercase sparingly — mainly for labels and subtitle bars
- Keep font sizes conservative in SVG (max 96px for hero, 64px for standard titles)
- Always leave 15-20% horizontal safety margin for monospace text fallback

## Layout Guidance

- Prefer strong left alignment
- Use ambient radial glows instead of heavy line grids for depth
- Add sparse dot grids for texture without clutter
- Keep screenshots inside dark frames for visual cohesion
- Leave generous negative space around the mark and tagline
- Consistent border radius: 36px outer, 24px inner, 14-18px pills

## Logo Anatomy

The mark consists of:
1. **Outer hexagon** — cyan gradient stroke, represents the Conflux network
2. **Inner hexagon** — dark fill, represents the local development space
3. **`<` chevron** — cyan, code/terminal identity
4. **`/` slash** — orange, file path / division
5. **`_` underscore** — white, cursor / active state
6. **Accent dot** — orange, at upper-right, the Conflux energy spark

## Messaging Pillars

1. **Faster setup** — under two minutes to a working environment
2. **Real node fidelity** — persistent chain data and resumable state
3. **AI-native workflows** — MCP tools exposed to assistants
4. **Full-stack local** — node, backend, DEX, templates, editor integration

## What to Avoid

- Generic coins, rockets, or glowing chains as primary symbols
- Glossy gradients without structural geometry
- Bright purple or neon rainbow palettes
- Overusing orange until it overwhelms the cyan/dev-tool identity
- Font sizes above 96px in any SVG asset (causes overflow on fallback fonts)
