# Screenshot Usage

Use `screenshot-frame.svg` when creating polished screenshot composites for:
- README hero sections
- Pitch deck feature slides
- Social announcement images
- Submission folder screenshots

## Workflow

1. Capture the raw screenshot at 1920x1080 or 2560x1440
2. Crop to the key UI area
3. Place inside the frame's dashed capture zone
4. Add 1-2 callout annotations (arrows, boxes) — do not over-annotate
5. Use cyan for primary callouts, orange only for a single contrast highlight

## Priority screenshots

From the submission plan:
1. Devcontainer start (VS Code loading the workspace)
2. Status bar showing stack running
3. Copilot deploying a contract via MCP tool calls
4. DEX UI with live pairs
5. Contract explorer tree view
6. MCP tools in action (bonus)
7. Scaffold CLI output (bonus)

## Consistency rules

- Use the same frame for all public-facing captures
- Prefer 16:9 or 16:10 source screenshots
- Crop only after the key UI element is clearly readable
- Keep dark theme in captures so they match the frame background
