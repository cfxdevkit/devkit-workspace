`pnpm smoke:workspace` is the intended project-level smoke command. It refreshes generated state, verifies script contracts, and then builds the dapp.

The generated devcontainer builds a tiny local `.devcontainer/Dockerfile` that extends the shared infrastructure image through a build arg defaulting to `new-devkit-devcontainer:dev`. Backend and VS Code extension infrastructure are baked into that reusable image rather than copied into each generated project. Build the shared image from the main workspace before opening the generated project in a devcontainer.

The shared image build also tags a canonical GHCR-style alias for future publishing, but local generated devcontainers intentionally default to the local tag to avoid registry lookups during normal local development.

Generated `devcontainer.json` files currently set `"pull": "never"` so Dev Containers does not attempt registry fallback before these images are published.

Recommended workflow:

1. run `pnpm run image:build:devcontainer` in the main `new-devkit-workspace`
2. run `pnpm run image:check:devcontainer` in the main `new-devkit-workspace`
3. regenerate the project scaffold for the devcontainer target
4. open the regenerated project folder in Dev Containers

If Dev Containers logs still show `docker pull new-devkit-devcontainer:dev`, the generated project is stale and still carries an older `.devcontainer/devcontainer.json`. Regenerate the scaffold before retrying.

## Agent-Friendly Script Contracts

- root utility scripts support `--json` for machine-readable output
- each utility script writes its last result into `.devkit/operations/<operation>.latest.json`
- each utility script appends an entry to `.devkit/operations/ledger.ndjson`
- generated state stays explicit in `dapp/src/generated/`
- editor infrastructure is shared through the target image instead of copied into project files
